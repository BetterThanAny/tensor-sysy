# W8 — CUDA Adapter + 单算子 Codegen · 设计文档

**日期**：2026-04-18
**项目**：tensor-sysy（WSL 环境延续 macOS 侧 W0–W7 成果）
**对应 PLAN.md 行**：§W8「CUDA Adapter + 单算子 Codegen」

## 1. 目标

对标 W7 CPU 闭环（`examples/mlp.tsy → tsc emit-cpp → ./build/out/mlp`），在 CUDA 路径上拿到 **单算子** 级别等价闭环：

- `tsc run-lir --backend=cuda-adapter foo.tsy` 能跑
- `tsc emit-cu foo.tsy → nvcc 编译 → 可执行二进制` 能跑
- CUDA 输出与 CPU adapter 输出按容差对拍通过

## 2. 范围

**全量 W8**（用户 2026-04-18 确认）：

- In-scope：`run-lir --backend=cuda-adapter`、`tsc emit-cu` 子命令、CUDA codegen 产物编译+运行、CPU/CUDA 对拍测试
- Out-of-scope：多算子 fusion、batched matmul、int8 量化、完整 MLP 的 CUDA codegen（W10）、scheduler + layout lowering（W9）、性能 benchmark（W11）

## 3. Kernel 源选择（方案 A）

用户 2026-04-18 确认：**自写 FP32 kernel**，不复用 `mini-llm-engine/src/ops_cuda.*`（纯 FP16）也不修改 `mini-llm-engine/cuda-kernels/*`（benchmark-style 不可链接）。

| 原 PLAN.md 假设 | 实际情况 | 本设计决策 |
|---|---|---|
| `mini-llm-engine/ops_cuda.cu` 有 `matmul_cuda` FP32 接口 | 实际是 `gemm_fp16` FP16 接口，且绑 `Tensor`/`Device`/`DType` 类 | 绕开，自写 FP32 adapter |
| 复用 mini-llm-engine 的 kernel | 复用会拖入 sentencepiece + FP16 精度损失（atol ≈ 1e-2）| tensor-sysy 侧自写 FP32 小 kernel + cuBLAS sgemm |

**tolerance**：FP32 cuBLAS vs CPU reference `atol=1e-4, rtol=1e-3`（PLAN.md §W8 测试方案给的值）。若实际跑下来 odd-shape 超 1e-4，退让到 `atol=5e-4, rtol=1e-3`。

## 4. 架构

```
.tsy source
   │ (flex/bison frontend)
   ▼
AST ─► HIR ─► LIR (所有层 FP32)
                │
                ├── run-lir --backend=native       → W4 naive interpreter
                ├── run-lir --backend=cpu-adapter  → W6 CPU adapter
                ├── run-lir --backend=cuda-adapter → W8 NEW: adapter_cuda
                │                                       ├─ cuBLAS sgemm
                │                                       └─ 手写 softmax/rmsnorm/add kernel
                │
                └── emit-cpp / emit-cu
                      ├── emit-cpp  → W7 CPU 宿主源
                      └── emit-cu   → W8 NEW: CUDA 宿主源（.cu，含 main()）
                                        链接 tsy_runtime_cuda → nvcc 编译
```

每层职责保持和 W0–W7 一致：LIR 以上完全不感知 CUDA 存在；CUDA 路径只在 runtime layer 和 codegen 分支出现。

## 5. 组件详表

### 5.1 `src/runtime/adapter_cuda.{h,cu}` (NEW)

和 `adapter_cpu.{h,cpp}` 严格对称。命名空间 `tsy::runtime`。每个 primitive 自己管 host↔device 拷贝（简单、无状态）。

```cpp
// 入口，签名与 CPU 版对称
void adapterMatMulCuda(const Tensor& a, const Tensor& b, Tensor& c);
void adapterAddCuda   (const Tensor& a, const Tensor& b, Tensor& c);
void adapterSoftmaxCuda(const Tensor& x, Tensor& y);
void adapterRMSNormCuda(const Tensor& x, Tensor& y);

// Executor（和 runWithCpuAdapter 对称）
tsy::lir::RunResult runWithCudaAdapter     (const tsy::lir::Module&, tsy::DiagnosticEngine&);
tsy::lir::RunResult runNamedWithCudaAdapter(const tsy::lir::Module&, const std::string&,
                                             tsy::DiagnosticEngine&);
```

**matmul 的 cuBLAS 列主/行主处理**：tensor-sysy 和 CPU adapter 都是行主 `C[M,N] = A[M,K] @ B[K,N]`。cuBLAS 默认列主，用恒等式 `row-major C = A @ B` ≡ `col-major C^T = B^T @ A^T`，故调用
```
cublasSgemm(h,
            CUBLAS_OP_N, CUBLAS_OP_N,
            N, M, K,
            &alpha,
            B_dev, N,   // "B in col-major = B^T in row-major"
            A_dev, K,
            &beta,
            C_dev, N)
```
不预转置、不 materialize 中间缓冲。这与 mini-llm-engine/ops_cuda.cu 的 `gemm_fp16` 使用同一恒等式。

**softmax**：一个 block 一行，block=256 threads，two-pass max-reduce → exp-sum-reduce → normalize，shared mem 做 reduce。

**rmsnorm**：一个 block 一行，block=128 threads (4 warps)，warp-shuffle reduce 求 `sum(x^2)`，cross-warp smem，权重恒等于 `ones`（HIR 目前不暴露 gain vector，和 CPU adapter 一致），eps=1e-6。

**add**：flat elementwise，`blocks = ceil(n / 256)`。

**ones-weight device cache**（rmsnorm 用）：和 CPU 的 `onesVector()` 镜像，按 `inner_dim` 缓存一份 device buffer。第一次 call 时 `cudaMalloc + cudaMemset` fill 1.0。

### 5.2 `src/codegen/cuda.{h,cpp}` (NEW)

和 `codegen/cpp.{h,cpp}` 对称。入口：
```cpp
namespace tsy::codegen {
void emitCudaModule(std::ostream& out, const tsy::lir::Module& m, const std::string& src);
}
```

产出 `.cu` 文件结构：
```cpp
// AUTO-GENERATED by `tsc emit-cu`. Do not edit.
// Source: examples/matmul_tiny_cuda.tsy
#include <cstdint>
#include <initializer_list>
#include <iostream>
#include <vector>
#include "adapter_cuda.h"
#include "interpreter.h"   // NamedTensor / fillDeterministic / print helpers

namespace { /* makeBuf / printTensor / ... 复用 W7 头 */ }

int main() {
    /* 为每个 buffer 分配 NamedTensor，fillDeterministic 参数
       对 call 指令，调 adapterXxxCuda
       print 和 CPU 版同文本布局 */
}
```

`tsc emit-cu` CLI：
```
tsc emit-cu <foo.tsy> [-o <out.cu>]
```

默认 stdout 输出；`-o` 指定路径。复用 `emit-cpp` 的 `Options.output_path`。

### 5.4 `CMakeLists.txt` 扩展

```cmake
include(CheckLanguage)
check_language(CUDA)
if(CMAKE_CUDA_COMPILER)
    enable_language(CUDA)
    set(CMAKE_CUDA_STANDARD 17)
    set(CMAKE_CUDA_ARCHITECTURES 86)        # RTX 3080 / Ampere
    set(CMAKE_CUDA_RUNTIME_LIBRARY Shared)

    # WSL2 libcuda stub 修复（搬自 mini-llm-engine/CMakeLists.txt）
    if(EXISTS "/usr/lib/wsl/lib/libcuda.so.1")
        set(CMAKE_BUILD_RPATH "/usr/lib/wsl/lib")
        add_link_options("LINKER:--disable-new-dtags")
        set(CMAKE_EXE_LINKER_FLAGS
            "${CMAKE_EXE_LINKER_FLAGS} -Wl,--no-as-needed -L/usr/lib/wsl/lib -lcuda -Wl,--as-needed")
    endif()

    find_package(CUDAToolkit REQUIRED)

    add_library(tsy_runtime_cuda STATIC src/runtime/adapter_cuda.cu)
    target_include_directories(tsy_runtime_cuda PUBLIC
        ${CMAKE_CURRENT_SOURCE_DIR}/src/runtime)
    target_link_libraries(tsy_runtime_cuda PUBLIC
        tsy_lir CUDA::cudart CUDA::cublas)

    set(TSY_HAVE_RUNTIME_CUDA 1)
    message(STATUS "tsy_runtime_cuda: enabled (arch sm_86)")
else()
    set(TSY_HAVE_RUNTIME_CUDA 0)
    message(STATUS "tsy_runtime_cuda: CUDA not detected, CUDA tests will be skipped")
endif()

# tsc 条件链接 cuda adapter
if(TSY_HAVE_RUNTIME_CUDA)
    target_link_libraries(tsc PRIVATE tsy_runtime_cuda)
    target_compile_definitions(tsc PRIVATE TSY_HAVE_RUNTIME_CUDA=1)
endif()

# tsy_add_cuda_example helper（对应 tsy_add_example）
function(tsy_add_cuda_example NAME TSY_FILE)
    if(NOT TSY_HAVE_RUNTIME_CUDA)
        message(STATUS "tsy_add_cuda_example(${NAME}): skipped (no CUDA)")
        return()
    endif()
    set(_src "${CMAKE_CURRENT_SOURCE_DIR}/${TSY_FILE}")
    set(_gen "${CMAKE_CURRENT_BINARY_DIR}/gen/${NAME}.cu")
    add_custom_command(
        OUTPUT ${_gen}
        COMMAND ${CMAKE_COMMAND} -E make_directory ${CMAKE_CURRENT_BINARY_DIR}/gen
        COMMAND $<TARGET_FILE:tsc> emit-cu ${_src} -o ${_gen}
        DEPENDS tsc ${_src}
        COMMENT "tsc emit-cu ${TSY_FILE} -> gen/${NAME}.cu"
        VERBATIM)
    add_executable(${NAME} ${_gen})
    set_source_files_properties(${_gen} PROPERTIES LANGUAGE CUDA)
    target_link_libraries(${NAME} PRIVATE tsy_runtime_cuda)
    set_target_properties(${NAME} PROPERTIES
        RUNTIME_OUTPUT_DIRECTORY "${CMAKE_CURRENT_BINARY_DIR}/out"
        CUDA_ARCHITECTURES 86)
endfunction()

tsy_add_cuda_example(matmul_cuda_demo examples/matmul_tiny_cuda.tsy)
```

`tsy_codegen` 新增 `src/codegen/cuda.cpp`。

## 7. 风险 + 缓解

| 风险 | Mitigation |
|---|---|
| cuBLAS 列主/行主恒等式用错，矩阵乘结果转置 | W8 一开始先写 `matmul 2×2×2` tiny smoke，和 CPU 对拍；错了立刻感知 |
| `sm_86` 在某些 toolkit 组合下 unsupported | `check_language(CUDA)` 做 configure-time fallback；另外 12.0 toolkit 官方支持 sm_86 |
| WSL libcuda stub 导致 runtime load 失败 | DT_RPATH 片段已验证在 mini-llm-engine 工作；照搬 |
| FP32 容差在 odd-shape / 长条 超 1e-4 | PLAN.md 已允许微调，fallback `atol=5e-4, rtol=1e-3` |
| emit-cu 生成的 .cu 和 `adapter_cuda.h` 头路径不一致 | codegen 里用相对 include，配合 CMake `target_include_directories` |
| Ones-weight device cache 线程安全 | W8 单线程 executor，不用锁；有 mutex 注释表明未来并发时需加 |
| 单算子 codegen 测试被 ctest 时间拖累 | fixture 用现有 2×2 shape，单个测试 <100ms |
